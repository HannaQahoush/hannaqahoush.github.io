Mini Query Execution Engine

A custom SQL query execution engine built in Java that simulates how a database management system processes queries internally, using an iterator-based pipeline model to evaluate and compare three join algorithms.

Project Overview

The engine follows the iterator-based model — a pipeline where each operator processes tuples one at a time and passes results to the next operator in the chain. Every operator implements a single getNextTuple() interface method, keeping the architecture modular and composable.

The engine parses a SQL query, builds an operator tree (Scan → Selection → Join → Projection), executes it, and reports execution time, block accesses, and row count for all three join algorithms simultaneously.

Projection

Sits at the top — outputs only the columns requested in SELECT

check_circle

NestedLoopJoin

Loads right relation into ArrayList, scans all right tuples per left tuple — O(N×M)

check_circle

HashJoin

Builds a HashMap from right relation, O(1) lookup per left tuple — O(N+M)

check_circle

MergeJoin

Sorts both relations then merges with two pointers — O(N log N + M log M)

check_circle

Tuple

Represents a single record; implements Comparable to support sorting by id

Dataset

Two CSV files of 10,000 records each with randomly distributed data. Block size set to 1024 bytes (~10 tuples per block, unspanned format).
