CPU Scheduling Evaluation

Implementation and performance comparison of five CPU scheduling algorithms — FCFS, SRTF, SJF, Round Robin, and Priority Scheduling — evaluated on average waiting time and turnaround time.

View Source on GitHub open_in_new

Project Overview

CPU scheduling determines which process the operating system assigns to the processor and for how long when multiple processes compete for CPU time. The scheduler's goal is to maximize CPU utilization while minimizing waiting time and turnaround time for all processes.

This project implements five scheduling algorithms from scratch and runs them against an identical set of test processes, measuring average waiting time and average turnaround time for each to draw a fair performance comparison.

Algorithms Implemented

First-Come First-Served (FCFS)

Non-Preemptive

Processes execute in arrival order. Simple but can cause the convoy effect where short processes wait behind long ones.

Shortest Remaining Time First (SRTF)

Preemptive

Preemptive version of SJF. On each new arrival, compares burst time to the running process and preempts if shorter. Uses a priority queue keyed by remaining time.

Shortest Job First (SJF)

Non-Preemptive

At each decision point, picks the arrived process with the smallest burst time. Optimal for minimizing average waiting time but requires knowing burst time in advance.

Round Robin (RR)

Preemptive

Each process gets a fixed time quantum (3 units). If not finished, it's placed back at the end of the circular queue. Guarantees fairness for interactive systems.

Priority Scheduling

Non-Preemptive

Each process has a numerical priority; lower value = higher priority. Risk of starvation for low-priority processes if high-priority processes keep arriving.

Key Takeaways

emoji_events

SJF is optimal for waiting time

Always picks the shortest available job, minimizing time other processes spend waiting — but requires knowing burst time in advance.

info

Round Robin trades efficiency for fairness

Highest waiting time due to quantum overhead, but guarantees every process gets CPU time — ideal for interactive and time-sharing systems.

warning

No universally best algorithm

The right choice depends entirely on the system's requirements, workload type, and whether burst times are known in advance.
